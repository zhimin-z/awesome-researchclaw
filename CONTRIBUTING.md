# Contribution Guidelines

Thank you for your interest in contributing to Awesome ResearchClaw! 🦞

## Adding a Project

Please ensure your pull request meets the following criteria:

- **Open source or source-available** — the project must have a public repository.
- **Actively maintained** — no dead repos with no maintenance signal.
- **Relevant** — related to ResearchClaw, research automation, AI-powered academic workflows, or the OpenClaw ecosystem.
- **Useful in practice** — not just a demo; should have a working installation, docs, or screenshots.
- **Description** — add a short description focused on **what job the project is useful for**.

## Format

Use the following format:

```markdown
- [Project Name](https://github.com/user/repo) — Short description of what it does and why it's useful.
```

## Pull Request Process

1. Search previous suggestions before making a new one.
2. Make an individual pull request for each suggestion.
3. Use the following format: `[Project Name](link) — Description.`
4. New categories or improvements to the existing categorization are welcome.
5. Keep descriptions concise and informative.
6. Check your spelling and grammar.
7. Make sure your text editor is set to remove trailing whitespace.

## Suggested Exclusions

- Closed-source SaaS with no public code
- Dead repos with no maintenance signal
- Generic AI lists with no relation to the ResearchClaw / research automation ecosystem
- Low-signal clones with unclear authenticity

Thank you for your suggestions! 🙏
